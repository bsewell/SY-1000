# Handoff ‚Äî 2026-03-31-08

## Context

Session driven by an external developer attempting to build the project on Windows, Linux,
and macOS Intel. They hit three issues: missing `stamp_version.sh`, BSD/GNU `sed` incompatibility
on Linux, and `backtrace()` / POSIX-only headers failing to compile on Windows.

---

## Changes made this session

### 1. `tools/stamp_version.sh` ‚Äî added to git
- Was present on disk but excluded from version control
- Any fresh clone was missing it, causing `./build.sh` to fail immediately on macOS

### 2. `.gitignore` ‚Äî removed `build.sh`
- `build.sh` was gitignored, meaning clones did not include it
- External developers had no build script at all

### 3. `build.sh` ‚Äî Linux/macOS portability
- Added OS detection via `uname -s`
- `sed -i ''` ‚Üí `sed -i` on Linux (GNU sed does not accept empty-string argument)
- `sysctl -n hw.logicalcpu` ‚Üí `nproc` on Linux
- macOS app bundle deploy step wrapped in `[ "$OS" = "Darwin" ]` guard
- macOS behaviour unchanged

### 4. `tools/stamp_version.sh` ‚Äî Linux/macOS portability
- Same BSD vs GNU `sed` fix
- MyInfo.plist stamping wrapped in `[ "$OS" = "Darwin" ]` guard (plist is macOS app bundle artefact)

### 5. `tools/build.bat` ‚Äî new file (Windows)
- PowerShell-based version stamp (reads VERSION from `.pro`, increments build number)
- Auto-detects `nmake` (MSVC) vs `mingw32-make` (MinGW)
- Runs `qmake` then `make`
- Does not attempt app bundle deploy (Windows uses the `.exe` directly)

### 6. `src/main.cpp` ‚Äî Windows/Linux portability
All changes are additive `#ifdef` guards. macOS code path is byte-for-byte identical to before.

| Change | Detail |
|--------|--------|
| POSIX headers guarded | `execinfo.h`, `unistd.h`, `fcntl.h` behind `#ifndef _WIN32` |
| Windows headers added | `<windows.h>`, `<dbghelp.h>`, `<direct.h>`, `<io.h>`, `<process.h>` |
| POSIX name mapping | `getpid`, `getcwd`, `mkdir`, `popen`, `pclose`, `open`, `write`, `close` mapped to Windows `_`-prefixed equivalents |
| Crash handler | POSIX signal handlers replaced with `SetUnhandledExceptionFilter` + `MiniDumpWriteDump` on Windows |
| `SIGBUS` guarded | Not defined on Windows; wrapped in `#ifdef SIGBUS` |
| `configureQtPluginPaths` | Entire function body wrapped in `#ifdef Q_OS_MAC` (uses `otool -L`, macOS-only) |
| `shellEscapeDoubleQuoted` | Wrapped in `#ifdef Q_OS_MAC` to suppress unused-function warning on other platforms |
| `close()` return type | Fixed wrapper to return `int` (was incorrectly returning `void`) |
| `mkdir` consistency | Windows branch of `initCrashLog` was calling `_mkdir` directly; changed to use the `mkdir(p,m)` macro consistently |
| `setenv` ‚Üí `qputenv` | Cross-platform Qt equivalent for setting env vars before `QApplication` |
| Log directory | Windows uses `%APPDATA%\Gumtown\SY-1000FloorBoard\`; POSIX uses `~/Library/Application Support/...` |

## Known issues / watch items

- **Linux log path** ‚Äî on Linux, startup.log goes to `~/Library/Application Support/...` (macOS convention). This is non-standard on Linux (XDG would use `~/.local/share/...`). Functionally harmless but a real developer may want to add `$XDG_DATA_HOME` support.
- **`build.bat` has no errorlevel check on PowerShell calls** ‚Äî if the PowerShell version-stamp step fails silently, the build continues with a wrong version number. Low priority for now.
- **`qml-ui-migration` needs to pick up main** ‚Äî before the next QML panel work, rebase or merge main into `qml-ui-migration` so it has the portability fixes.
