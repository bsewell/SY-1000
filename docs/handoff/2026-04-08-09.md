## Changes Made This Session

### 1. `qml/ParaEqGraph.qml` — Three QML EQ bug fixes (builds 13-16)

Ported three fixes from `src/customParaEQGraph.cpp` to the QML reimplementation:

**Fix 1 — Node tracking (lines ~230-237)**
Changed from strict threshold (`Math.abs(...) < pw / 160.0`) to closest-point
tracking. When the mid-band center falls between sample points, the old code left
nodes undefined, causing null-reference crashes.

**Fix 2 — High cut inversion (line ~198)**
SY-1000 high cut is inverted: value 0 = max cut, 30 = flat. QML was not inverting.
Added `1.0 - highCut / 30.0` to match C++ behavior.

**Fix 3 — Drag bounds clamping (lines ~321-324)**
Added `Math.max/Math.min` clamping on mouse coordinates during drag, preventing
out-of-range values when dragging outside the widget (Windows platform issue).

### 5. `src/floorBoard.cpp` — Signal chain column alignment fix (builds 18-21)

**Root cause 1 — flowStep mismatch (line 1679):**
`ChainLayout` used `qRound(55.0*ratio) = 61` but `update_structure` used `int(55*ratio) = 60`.
Balancers (positioned via `layout.colX()`) used 61px columns while FX blocks advanced at 60px,
causing cumulative drift. Fixed: `const int flowStep = qRound(55.0*ratio);`

**Root cause 2 — compactSegmentAfterBalancer (lines 2244-2246):**
Post-layout compaction shifted entire segments after each balancer to close gaps, breaking
cross-row column alignment. FX3/Comp were shifted -14px left of FX1/FX2. Disabled:
the Boss reference layout uses strict column alignment with no per-segment compaction.

**Result:** All FX blocks now have zero column error. Cross-row alignment confirmed:
FX1=FX3=Comp at x=316, Bal1=Amp at x=438, Bal2=NS at x=560.
