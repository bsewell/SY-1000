## Known Issues / Bugs Found

### CRASH: `SysxIO::updatePatch` — indexOf returns -1, corrupts rebuild string

**Crash report version:** 2026.04.02.12 (older build, but bug still exists in current code)

**Root cause:** `src/SysxIO.cpp` lines 1814 and 1855:
```cpp
rebuild.replace(rebuild.indexOf("F041"+unitID+"0000006912", nextIndex)+16, 4, "10"+addr1);
```
If `indexOf` returns -1 (pattern not found in MIDI reply), the expression becomes `rebuild.replace(15, 4, ...)` — corrupting the sysex header at position 15. This corrupted string is then parsed by `setFileSource("Structure", rebuild)`, which creates invalid QStrings that crash when `QList<QString>::replace()` tries to assign them.

**Trigger:** Receiving a MIDI patch reply where the sysex header pattern `F041<unitID>0000006912` doesn't appear the expected 172 times (guitar) or 169 times (bass). Could happen with truncated MIDI data, unexpected unit ID, or communication errors.

**Affected lines:**
- `SysxIO.cpp:1814` — Guitar mode rebuild loop (172 iterations)
- `SysxIO.cpp:1855` — Bass mode rebuild loop (169 iterations)

**Secondary risk:** `SysxIO.cpp:254` — `data.at(i+1)` in the 2-arg `setFileSource` has no bounds check; can read past end of string.

**Status:** NOT FIXED. Analysis complete, fix not yet implemented.
