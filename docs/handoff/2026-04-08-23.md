## Actual Codebase Findings (correct target, not yet fixed)

### The FloorBoard signal chain layout lives in:
- **`src/chainLayout.h`** — `ChainLayout` struct with 7-row grid (`ChainRow` enum), `colX()`, `rowCenterY()`
- **`src/chainLayout.cpp`** — Constructor computes row centres and column positions
- **`src/floorBoard.cpp` lines 1554–1967** — `update_structure()` — the main layout function

### How the FloorBoard layout actually works:
1. Reads MIDI chain data to get block counts per zone (`index1`..`index6` = blocks on each input branch, `index7`/`index8`/`index9` = DIV/MIX positions)
2. Computes balancer column positions: `bal1xpos`, `bal2xpos`, `bal3xpos` with `MAX_BAL_SPREAD=2` capping (§L1 rule)
3. Places blocks sequentially with `x_axis += flowStep` per block
4. Splitter markers (`stomp>27`) trigger row transitions via `incr` counter (0=INST1, 1=BAL1Out, 2=INST3, 3=BAL2Out, 4=NORMAL, 5=BAL3Out)
5. DIV/MIX branches use `ab_y_axis`, `ab_start`, `ab_end` for dual-channel vertical offset

### Key layout constants (from `chainLayout.h`):
- `kRowSpacing = 50.0` (vertical distance between instrument rows)
- `kFlowStep = 48.0` (horizontal column pitch)
- `kTouchGap = 8.0` (gap between source block and first FX)
- `kInstStartX = 15.0` (source block left edge)

### Column alignment already has validators:
- `[COL-CHECK]` debug output groups blocks by column X and flags misalignment
- `[GAP-CHECK]` flags excessive inter-block gaps
- `[LAYOUT §L1]` logs balancer capping

### DIV/MIX vertical lines already use stem extension:
- `branchStemExtension = qRound(10 * ratio)` — extends lines above/below the block
- `topOffsetYForId()` returns `-branchStemExtension` (above block top)
- `bottomOffsetYForId()` returns `flowRect.height() + branchStemExtension` (below block bottom)
- Polygon points 14/15 (above DIV/MIX) and 16/17 (below DIV/MIX) use these offsets

## Known Issues

### Column misalignment in "Dave mix" preset
The user reports columns beyond column 1 are not being respected. The layout code in `update_structure()` places blocks per-row using `x_axis += flowStep`, and row transitions reset `x_axis` to either `firstFlowX` or `firstFlowX - flowStep`. Blocks at the same relative position on different rows should share column X values, but the `leftFlowOffsetXForId()` adjustment (line 1835) varies per stomp type, which could shift individual blocks off their column grid.

### DIV/MIX signal path not connecting to circle edges
The user reports vertical lines should emerge from the very top/bottom of the DIV/MIX circles. The `branchStemExtension` (10px * ratio) extends lines beyond the block bounds, but the polygon points for above/below DIV/MIX (indices 14-17) depend on `dividerTopY`/`dividerBottomY`/`mixerTopY`/`mixerBottomY` which are computed from `topOffsetYForId()`/`bottomOffsetYForId()`. These may not align with the actual circle edges in the rendered art.
