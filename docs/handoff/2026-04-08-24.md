## Changes Made

### `src/floorBoard.cpp` — Signal chain polygon math and branch wire logic

**Fix 1: Polygon wire left-endpoint magic numbers (around line 1991)**
- **Problem:** INST1/INST2 wire starts used hardcoded `60*ratio` and INST3/NORMAL used `78*ratio`. These values were stale from a previous layout configuration and inconsistent across rows.
- **Fix:** Replaced all four with a single computed value: `instRightEdgeX = offset + instStartX + instWidth` — the actual right edge of the source block at the current scale.
- **Before (ratio=1):** Wire starts at x=60 (INST1/2) or x=78 (INST3/NRM) — both inside the 80px source block at wrong positions
- **After (ratio=1):** Wire starts at x=95 — correct right edge of source block

**Fix 2: Polygon wire right-endpoint drift (around line 1992)**
- **Problem:** Right endpoints for INST1/INST2 branch wires used `105*ratio + index*flowStep`. The constant `105` was stale; the correct value is `firstFlowX` (103 at ratio=1). This caused a 2px drift that grew at higher scale ratios.
- **Fix:** Replaced `105*ratio` with `firstFlowX` so right endpoints exactly match `layout.colX()` grid positions. Now polygon[1].x == polygon[4].x (BAL1 riser is pixel-perfect).

**Fix 3: Generalized branch wire refinement (around line 2128)**
- **Problem:** The branch wire right-edge refinement only checked FX1 (id=8), FX2 (id=9), FX3 (id=10) by hardcoded ID. If other effect types (EQ, DS, AMP, etc.) occupied INST rows, the refinement didn't fire and wires used the fallback initial values.
- **Fix:** Added per-row rightmost-block tracking (`lastStompOnInst1`, `lastStompOnInst2`, `lastStompOnInst3`) during the placement loop. The refinement now uses whichever block is actually rightmost on each row, regardless of effect type.
