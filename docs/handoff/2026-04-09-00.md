## Changes Made

### `src/floorBoard.cpp`

**1. `placeBalancerOnRiser` — removed X collision avoidance (lines ~2199-2226)**

The lambda previously scanned all nearby blocks and nudged the balancer circle left/right to avoid overlaps. This pushed BAL1/2/3 off the column grid, breaking cross-row alignment. The first pass already places balancers at correct grid columns (`layout.colX(bal*xpos)`), and the 13px inter-column gap (flowStep 48 - block width 40 at ratio=1) provides enough clearance.

Fix: the lambda now only computes the Y midpoint (riser center) and keeps the X from the polygon point set in the first pass. No collision avoidance.

**2. `repositionBALOutputBlocks` — grid-based startX (lines ~2252-2300)**

Previously computed `startX = balRightX + standardFxGap` (balancer's visual right edge + gap). When `placeBalancerOnRiser` moved the balancer off-grid, all subsequent Row B blocks inherited the offset.

Fix: now takes a `balColIndex` parameter and computes `startX = this->offset + firstFlowX + (balColIndex + 1) * flowStep`. Call sites pass `bal1xpos`, `bal2xpos`, `bal3xpos` respectively.

**3. DIV/MIX wire rendering — straight-through connection (lines ~266-270)**

Previously drew a 4-line rectangle (A path top, B path bottom, left riser, right riser) around the DIV/MIX area, with dual/single channel variants showing grey dashed inactive paths.

Fix: replaced with a single straight line from polygon[13] (DIV center) → polygon[18] (MIX center), matching Boss Tone Studio which shows DIV and MIX as inline nodes with a clean horizontal connection.

**4. Temporary debug logging (lines ~1526, ~1860)**

Added `fprintf` debug output to `/tmp/sy1000_chain_debug.txt` that dumps first-pass block positions, grid offsets, and column indices on every `update_structure()` call. **Must be removed before release.**
