### 2. SyDataKnob — multi-nibble parameter support for QML

**New file**: `qml/SyDataKnob.qml`

QML knob component for parameters that span multiple SysEx nibble-bytes (e.g., delay time 0-2000ms stored as 4 hex digits across 4 consecutive addresses). Mirrors the legacy `customDataKnob` C++ widget behavior.

**Modified file**: `src/parameterBridge.h` / `src/parameterBridge.cpp`

Added 6 new `Q_INVOKABLE` methods:
- `getDataValue(hex0, hex1, hex2, hex3, dataType)` — reads N consecutive nibble-bytes, combines into int
- `setDataValue(hex0, hex1, hex2, hex3, dataType, value)` — splits int into nibbles, writes N bytes
- `getDataMin(dataType)` / `getDataMax(dataType)` — range from Tables section of midi.xml
- `getDataLabel(dataType)` — label from Tables
- `getDataDisplayValue(dataType, value)` — formatted display text from Tables

Added private helper: `resolveDataType(dataType)` maps keyword strings to Tables hex address and byte count:
- `"DELAY2000"` → Tables/00/00/0D, 4 bytes
- `"DELAY1300"` → Tables/00/00/0B, 3 bytes
- `"PREDELAY"` → Tables/00/00/07, 4 bytes
- `"BPM"` → Tables/00/00/0A, 4 bytes
- Plus 7 more types matching `customDataKnob.cpp` lines 51-63

## Key Technical Notes for Next Session

- **Nibble encoding**: Roland SysEx stores multi-byte values as individual hex digits in consecutive addresses. Value 500 → "01F4" → bytes [00,01,0F,04] at hex3, hex3+1, hex3+2, hex3+3.
- **Tables lookup**: `customDataKnob` and `SyDataKnob` use the "Tables" root in midi.xml for range/display, NOT the direct parameter address. The `resolveDataType()` maps keyword → Tables hex_c.
- **QRC cache**: Always bust with `rm -f build/release/qrc_qml.o qrc_qml.cpp` before building when QML files change.
- **Diagnostic server**: TCP `localhost:47321`. Commands: `chain-positions`, `inspect-edit-panel`, `select-stomp N`, `screenshot PATH`.
