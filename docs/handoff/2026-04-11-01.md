# Handoff — 2026-04-11-01

## Summary

Added **MIDI CC controller input support** to SY-1000 FloorBoard. External MIDI controllers (Launch Control XL, Panorama CS12R, MPK Mini MK4, etc.) can now drive on-screen knobs and send SysEx to the SY-1000 in real-time. Includes a live MIDI monitor dock panel and a dedicated CC controller port (separate from the SysEx port).

## Architecture

```
Launch Control XL (USB)
  └─ midiCCIn (RtMidiIn #2, dedicated)
       └─ pollCCInput() [called every 10ms from getData() timer]
            └─ emit ccReceived(channel, ccNumber, value)
                 ├─ MidiCCHandler::handleCC()
                 │    └─ lookup cc_mappings.json
                 │    └─ scale 0-127 → param min/max
                 │    └─ ParameterBridge::setValue()
                 │         └─ SysxIO::setFileSource() → SysEx to SY-1000
                 │         └─ emit parameterChanged() → UI knob updates
                 └─ MidiMonitor::logCC() → visual log

SY-1000 (USB)
  └─ midiin (RtMidiIn #1, original)
       └─ SysEx communication unchanged
```

## Known Issues

1. **Single CC per poll cycle**: `pollCCInput()` reads one message per timer tick (10ms). Fast knob sweeps may drop messages. Could switch to a drain loop (`while (getMessage())`) for higher throughput.
2. **No "None" close-port logic**: When switching CC controller from a device back to "None", the `midiCCIn` port stays open until the next `pollCCInput()` call finds port -1. Should explicitly close on preference change.
3. **CC from SysEx port**: The original `getMsg()` also emits `ccReceived` for CCs arriving on the SysEx port. This is intentional (catches CCs from SY-1000 itself) but could cause duplicate events if both ports receive from the same device.
